# Agent Instructions

This repository contains the source code and content for a personal blog.

## Purpose of This Repository
- Publish blog posts and static pages
- Maintain site configuration, styling, and build logic
- Keep content readable, consistent, and easy to maintain

## What Agents May Do
- Add or edit blog posts
- Fix typos, broken links, or formatting issues
- Refactor or clean up code when explicitly requested
- Update configuration files when required for new features

## What Agents Must Not Do
- Change the blog’s theme, layout, or visual design unless explicitly asked
- Rewrite existing posts for tone or opinion without permission
- Introduce new dependencies without justification
- Remove content unless instructed

## Content Guidelines
- Preserve the author’s voice and intent
- Prefer clarity over cleverness
- Avoid unnecessary jargon
- Use consistent formatting and front-matter fields

## Writing Conventions
- Use Markdown for all posts
- One post per file
- Filenames should be URL-safe and descriptive
- Keep line lengths reasonable for readability

## Code Conventions
- Follow existing project structure and patterns
- Prefer simple, explicit solutions
- Avoid premature optimization

## Commit / Change Style
- Make minimal, focused changes
- Do not mix unrelated edits in a single change
- Explain non-obvious decisions in comments

## Questions & Ambiguity
If instructions are unclear or multiple valid approaches exist:
- Ask for clarification before making large or irreversible changes

## Assumptions
- This is a personal blog, not a commercial product
- Stability and maintainability are more important than novelty
